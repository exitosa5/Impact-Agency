## INPUTS DEL CLIENTE

```
Cliente: [nombre del negocio]
Sector: [restaurante / hotel / antro / inmobiliaria / otro]
Servicio a anunciar: [qué se está promoviendo]
Palabra clave principal: [término de búsqueda objetivo]
Ubicación geográfica: [ciudad / zona]
CTA deseado: [llamar / reservar / cotizar / visitar / descargar]
URL de destino: [landing page o página de destino]
```

---

## ESTRUCTURA RSA (Responsive Search Ad)

### Titulares (máx. 30 caracteres c/u — entregar 10–15 opciones)

Google combina automáticamente los titulares en pares. Escribir opciones que funcionen solas y en combinación.

**Grupos de titulares:**

Grupo A — Propuesta de valor:
```
1. [beneficio principal en ≤30 chars]
2. [diferenciador en ≤30 chars]
3. [resultado esperado en ≤30 chars]
```

Grupo B — Keyword + acción:
```
4. [keyword principal en ≤30 chars]
5. [keyword + ciudad en ≤30 chars]
6. [keyword + CTA en ≤30 chars]
```

Grupo C — Urgencia / social proof:
```
7. [número o dato social en ≤30 chars]
8. [urgencia o escasez en ≤30 chars]
9. [garantía o promesa en ≤30 chars]
```

Grupo D — Preguntas o hooks:
```
10. [pregunta que responde la búsqueda en ≤30 chars]
11. [variante de pregunta en ≤30 chars]
```

---

### Descripciones (máx. 90 caracteres c/u — entregar 4–6 opciones)

```
Desc 1: [beneficio + CTA — ≤90 chars]
Desc 2: [prueba social + CTA — ≤90 chars]
Desc 3: [oferta o urgencia + CTA — ≤90 chars]
Desc 4: [diferenciador + CTA — ≤90 chars]
```

## NOTA DE USO
- Usar titulares 1–6 como "fijados" en posición 1 y 2 para garantizar que la keyword y CTA aparezcan
- Rotar descripciones para A/B automático de Google
- Revisar los titulares ganadores en el informe de combinaciones cada 2 semanas
